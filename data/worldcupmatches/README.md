# `worldcupmatches` — labeled ground-truth dataset for WC2026

One canonical JSON record per World Cup 2026 match **we predicted**, written by the
feedback loop (`engine/feedback.py`) when a match finishes. Each record is a labeled,
out-of-sample example: the published prediction (frozen) + the realized outcome + the
proper scores comparing them. These accumulate into the dataset we mine for feature
validation (host advantage, xG-vs-goals) and the tournament performance monitor.

Records are committed (small JSON, text, diffable). `_monitor.{md,json}` is the rolled-up
view, regenerated by `engine.feedback.monitor()`.

## How a record is created

```bash
# the night the match ends — score it against what we published
python scripts/record_match.py predictions/ira-nor 0 2 --stage group --group E --monitor

# later, backfill rich stats from FBref (non-destructive)
python -c "from engine.feedback import backfill_rich; \
  backfill_rich('2026-06-16-ira-nor', xg_home=0.6, xg_away=2.3, possession_home=41)"
```

## Schema (`<match_key>.json`, `match_key = YYYY-MM-DD-<home3>-<away3>`)

Four layers that never mix:

- **`identity`** — date, home, away, neutral, venue, competition, stage, group.
- **`prediction`** — a FROZEN snapshot of what we published (ensemble + DC + Elo vectors
  in `[home, draw, away]` order, `params_hash`, as_of, xG, over2.5, btts, top scorelines).
  Scoring uses this, never a re-fit — a re-fit would leak the result through online-updated
  ratings and flatter the model dishonestly.
- **`outcome`** — ground truth, source-tagged and partially fillable:
  - `spine` — home_goals, away_goals, result (`home`/`draw`/`away`), source, captured_at.
  - `rich` — xg_home/away, shots, possession, ... + source, captured_at (backfilled later).
- **`scores`** — `y` (0=home,1=draw,2=away), `rps`, `log_loss`, `brier` (multiclass, 0..2),
  `surprisal` (−log p(actual)), and `per_model` scores for ensemble vs each component.

## Why this shape

- **Freeze the prediction** so every score is honest and reproducible (`params_hash`).
- **Provenance + partial fill**: live-night you have the score, not the xG — land the spine
  immediately, backfill rich stats later, each stamped with source and time.
- **Score the components too**: re-check "the ensemble beats DC/Elo alone" on real
  out-of-sample data instead of asserting it.
- **Surprisal** flags the confidently-wrong predictions — the bets worth a post-mortem.

See `design/design-step3-feedback-loop.md` for the full rationale.
